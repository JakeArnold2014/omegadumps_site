Privacy Policy for Omegadump Bot

Last updated: 26 June 2025

Omegadump Bot ("the Bot") is a tool designed to monitor and report item price drops ("dumps") in Old School RuneScape (OSRS). We respect your privacy and aim to keep our data handling minimal, transparent, and secure.

1. Information We Collect

Omegadump Bot does not collect or store personal information about users. However, in order to operate effectively, the Bot may process the following:

  • Discord Usernames/User IDs: Temporarily used for command handling and response.
  • Messages/Commands: Processed in real time; not stored beyond what’s necessary to respond.
  • Guild (Server) IDs: Used to manage command scope or per-server settings (if implemented).
  • Uploaded Files: Any files uploaded to the Bot may be stored for feature-related use (e.g. parsing item lists). These are not shared and are deleted when no longer needed.
  • OSRS Item Data: Pulled from public sources like the RuneScape Wiki or third-party APIs.

We do not collect passwords, emails, IPs, or sensitive identifiers.

2. Data Storage

  • Messages and user IDs are not stored permanently.
  • Uploaded files may be retained temporarily or until processing is complete, then deleted.
  • OSRS price data may be cached for performance, but contains no user info.
  • If future features introduce settings or user data storage, it will be opt-in and clearly documented.

3. Third-Party Services

The Bot may access external services including:

  • The RuneScape Wiki API for item and price data
  • Pastebin or static sources for item ID mapping

No personal data is sent to third parties.

4. User Controls

  • You can remove the Bot from your server at any time.
  • Uploaded files may be deleted upon request.
  • Contact the developer for questions or data handling concerns.

5. Changes to This Policy

We may update this policy to reflect new features or legal requirements. Significant changes will be announced via the support server or official channels.

6. Contact

If you have any questions or requests regarding this policy, contact the developer via the support Discord server or GitHub (if applicable).
